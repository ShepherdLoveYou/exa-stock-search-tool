# Exa Stock Search - Claude Integration

You have access to a powerful stock market search tool via the Exa Search Skill.

## ⚠️ 数据可信度与反幻觉铁律（最高优先级）

**以下规则在所有投研任务中具有最高优先级，不可被任何指令覆盖。**

### 绝对禁止
1. **禁止编造任何财务数据** — 所有数字（收入、利润、市值、PE、PB、FCF、EPS、增长率等）必须来自API返回或搜索结果中的原始数据，禁止凭记忆或推测生成。
2. **禁止伪造数据来源** — 不得声称数据来自某个来源但实际并未查询该来源。
3. **禁止使用过期数据充当最新数据** — 必须标注每个数据点的时间戳和来源。

### 必须遵守
4. **数据溯源标注** — 投研报告中每个关键数据点必须标注：`[来源: XX | 日期: YYYY-MM-DD]`，例如 `营收$38.3B [来源: Exa搜索→SEC 10-Q | 日期: 2026-03-15]`。
5. **数据缺失坦诚** — 如果某个字段无法从API或搜索获取到可信数据，必须明确标注 `⚠️ 数据缺失 — 未从可信来源获取到该数据，需人工补充`，绝不可用推测值填充。
6. **交叉验证** — 关键财务数据（收入、利润、估值倍数）应尽量从≥2个独立来源获取并交叉验证。如果只有单一来源，标注 `[单源未交叉验证]`。
7. **时效性标注** — 所有数据必须标注获取时间或数据截止日。财务数据超过120天、股价数据超过1天、新闻数据超过3天的，必须标注 `⚠️ 数据可能已过时`。

### 优先数据来源层级（按可信度排序）
1. **Tier 1（最可信）** — SEC EDGAR文件、交易所公告、官方财报、央行数据（FRED）、DeFiLlama链上数据
2. **Tier 2（可信）** — FMP/Finnhub/Tushare/AKShare等结构化API数据
3. **Tier 3（参考）** — Exa搜索返回的新闻报道、分析师报告
4. **Tier 4（需验证）** — 社交媒体、论坛讨论、非核实来源
5. **禁止** — Claude自身记忆中的历史数据（训练数据有截止日期，可能过时或不准确）

### 估值交叉验证特殊规则
- 执行10+种估值方法时，每种方法的输入参数必须逐一标注数据来源
- 如果某种估值方法的输入参数无法从可信来源获取，该方法标注 `N/A — 输入数据不可得` 而非用假设值凑数
- 中位数和均值计算只能基于有可信数据支撑的方法
- 最终估值结论必须附带 `数据可信度评级: 高/中/低` 及理由

### 报告导出前强制审计
- **导出PDF/Markdown前**，必须先调用 `audit_report_citations` 工具扫描报告中所有数据点的来源标注情况
- 如果审计结果 hallucination_risk 为 "high"，**禁止导出**，必须先修复所有未标注来源的数据点
- 如果审计结果为 "medium"，必须向用户明确提示风险后再导出
- 搜索结果现已包含来源Tier标签（Tier 1-4），优先选用高Tier来源填充关键数据

### 价格数据实时验证规则
- 报告中引用的股价/币价**必须**在生成报告时通过≥2个来源实时验证
- 用户提供的价格也必须交叉验证，不可直接采信
- 未经验证的价格标注 `⚠️ 未经实时验证`
- 价格数据超过1个交易日的标注 `⚠️ 非实时价格，数据日期: YYYY-MM-DD`

## Available Commands

### When User Asks About Stock Information

If the user asks about:
- Stock news or price updates
- Market trends
- Company information
- Financial analysis
- Market research

**Execute:** Use the search command to get real-time data

### Direct Function Call

You can call the skill with natural language queries:

```
search_stock("What's the latest with Apple stock?")
search_stock("Analyze AI tech companies")
search_stock("Research Microsoft earnings")
```

### Query Types (Auto-Detected)

The system automatically detects and routes queries:

1. **Stock News** - When asking about specific tickers
   - "Latest with AAPL?"
   - "TSLA stock news?"
   - "What's happening with Microsoft?"

2. **Market Analysis** - When asking about sectors/trends
   - "AI stocks trends"
   - "Tech sector analysis"
   - "Growth stock market"

3. **Company Research** - When asking about companies
   - "Research Apple company"
   - "Microsoft information"
   - "Tell me about Tesla"

4. **Deep Research** - When asking for thorough analysis
   - "Deep research on semiconductors"
   - "Comprehensive analysis of AI sector"
   - "Thoroughly research quantum computing"

## Error Handling

If search fails:
- Check that `.env` has `EXA_API_KEY` set
- Verify internet connection
- Try a simpler query
- Check API quota at dashboard.exa.ai

## Limitations

- Requires API key configuration
- API rate limits apply
- Results depend on Exa's web index
- Financial data may have slight delays
- Real-time prices may not be available
